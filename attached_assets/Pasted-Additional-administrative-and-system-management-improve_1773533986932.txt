# 15. Admin Module – User distribution and restaurant structure

The Users section in the Admin module needs to be redesigned in a more professional and structured way.

The system should support **clear operational hierarchy and assignment logic**.

### Accountant assignment

When creating or managing an accountant user, the system should allow assigning that accountant to specific restaurants.

Example:

Accountant → assigned restaurants list.

This allows the system to clearly define which restaurants the accountant is responsible for reviewing.

### Chief Accountant assignment

When creating a Chief Accountant, the system should allow assigning which accountants are managed under that Chief Accountant.

Example:

Chief Accountant → manages accountants A, B, C.

This hierarchy must be visible and editable from the admin interface.

# 16. Restaurant and brand structure management

Restaurant and brand structure must be managed professionally.

The system must support:

Brand
→ multiple restaurants / branches
→ restaurant-specific users

Each restaurant should have its own user assignments, such as:

• branch manager
• accountant (if applicable)
• operational roles

Users belonging to one restaurant should **not automatically appear in another restaurant unless assigned**.

The admin panel must allow:

• adding brands
• adding restaurants under each brand
• assigning users to each restaurant individually.
